    Part I. Preparing The PS3 and Install SACD-Ripper


    Warning: Doing the following steps will void your PlayStation 3 Warranty and I do not take and responsibilities for the damage it may cause to you and to your device. So, stay sharp and follow the instructions precisely and use common sense and your own good judgements to avoid unnecessary damages.

    MAKE SURE YOU UNPLUG YOUR PS3 FROM THE INTERNET AND DISABLE AUTOMATIC UPDATES BEFORE RESTORING CONNECTIVITY.
    Specifications about the SACD-Ripper Project can be found here:

    http://code.google.com/p/sacd-ripper/
    What you will need:

    First AND THE MOST

    First and the most, a Fat PlayStation 3 with SACD compatibility. How can you find out if your PS3 support SACD or not? Flip your PS3, and look for sign tags and if you PS3 support SACD, there will be a sign below:


    Here's also a nice chart provided by PS3SACD.COM that shows what specific model supports SACD.


    AND The PS3 has to have a firmware that is equal to or lower than 3.55 in order to successfully run the custom firmware.

    The bad stories like "Oh...I had it and my wife upgraded it..." can be heard all the time, but right now, there's not a simply and quick way to downgrade the PS3 since it is not designed to accept lower firmware versions.

    Once you are sure you have all the above specifications and correct firmware, read on.

    1. Get a USB Thumb drive. What size? Any thing bigger than 512MB will do for firmware flush and SACD-Ripper Installing. MAKE SURE IT'S FORMATTED AS FAT32.

    2. Unless you plan to rip entirely using the Daemon (as described below), you will need a storage device to plug in to the PS3 to store the rips. If you have a external hard drive, great, you're all set then. Note that the external hard drive will need a seperate power source - the USB ports on the PS3 will not provide sufficient power. If you don't have an external hard drive, it is recommended to get one or get a thumb drive that's bigger than 16GB to store the ripped SACD ISO or DSDIFF files.

    Get the Custom Firmware

    The kmeaw's 3.55 custom firmware is recommended for a better compatibility, although the App has been used succesfully on other custom firmwares, including OtherOS++ and Rebug.

    The already patched kmeaw's 3.55 CFW Patched PUP can be found here, download it and read on.

    IF YOU ARE RUNNING 3.54 OR LOWER

    1: Plug a USB stick into your computer
    2: Copy the custom firmware file(and rename it!) that you downloaded to a USB Thumb drive as /PS3/UPDATE/PS3UPDAT.PUP.
    3: Plug the USB stick into your PS3
    4: Navigate to Settings Tab
    5: Choose System Update
    6: Choose Update via Storage Media
    7: It will say it found Version 3.55
    8: Choose OK
    9: Accept Conditions and Follow the instructions[/quote]

    * Some users running 3.55 have been able to use this method to load a custom firmware like Rebug, which uses an artificially higher version number. However, the method below is recommended if you are running 3.55.

    IF YOU ARE RUNNING 3.55

    1: Plug a USB stick into your computer
    2: Copy the custom firmware file(and rename) that you downloaded to a USB thumb drive as /PS3/UPDATE/PS3UPDAT.PUP.
    3: Plug the USB stick into your PS3
    4: Power down the PS3 through the menu
    5: Now press and HOLD the power button, the system will startup and shutdownagain
    6: Release the power button, then press & HOLD power again, you'll hear one beep followed by two consecutive beeps
    7: Release power then follow the on-screen instructions. You're now in the recovery menu
    8: Connect the USB device and select "System Update."
    9: Accept Conditions and Follow the instructions[/quote]

    Once done, reboot and if everything went fine, you are now running 3.55 CFW. Congratulations!

    Now, you need to follow the instructions to install SACD-Ripper and finish setup.

    1. Download a copy of the ripper here: torrents.php?id=72186620
    2. Plug the thumb drive into computer
    3. put sacd-ripper.pkg and all the files under ps3keys into the root of the thumb drive.
    4. plug the thumb drive to PS3
    5. Under Games, choose Install Packages then SACD-Ripper.
    6. Upon finish the installation, run it!
    7. Follow the instructions and import the PS3 Keys.

    Once upon finish, you are now have a great SACD-Ripper! Cheers!

    SACD-Ripper supports the following output formats:

    - 2ch DSDIFF (DSD)
    - 2ch DSDIFF (DST) (if already DST encoded)
    - 2ch DSF (DSD)
    - mch DSDIFF (DSD)
    - mch DSDIFF (DST)
    - mch DSF (DSD)
    - ISO (due to the 4GB FAT32 size limit on the PS3, files will be splitted when larger)

    And...A little bit more notes from the developer:

    There seems to be a SACD BluRay read speed limit that sets the maximum read speed to 2.6MB/sec, yes I agree, this is slow! But DST decoding is even slower.. The PS3 GameOS has 6 SPUs available for user operations. SACD-Ripper uses 1 SPU for decryption and the other 5!! for DST decoding. Although this process is quite optimal the current decoding speed has been measured to be 0.9MB/sec for a 2 channel rip. So for 2ch DST tracks the write speed to disc will be around 2.3MB/sec, for a multi-channel the write speed is a bit higher at 2.4MB.sec but decoding is even slower at 0.6MB/sec.


    Part II. To Rip, To Convert and to Prepare for Upload
    Now, this is the part gets very interesting. I will split this into 2 parts to brief you through.

    The Basics
    1. You will need a external drive or a thumb drive that is larger than 8GB of space in order to save the ISO. The ISO usually runs about 3-5GB.
    2. The ISO that is larger than 4GB will be split to different files and you will need other software to combine the parts.
    3. The DSDIFF files are gigantic. A MultiChannel SACD can run somewhere from 5GB to 10GB just for MultiChannel DSDIFF files.
    4. The SACD-Ripper will log any read error occurred, so make sure you check it before you convert anything.
    5. Use "X" on the controller to submit and use "O" to navigate through the rip modes. (Stereo/MultiChannel DSD, DST, DSF or ISO)
    6. If not careful, you will overwrite the already ripped files since the naming is exactly the same. (Especially with 2 or more Disc sets) So, when you're done with one of the discs in the set, unplug it and export it first!
    7. if plugin your External Storage before insert SACD, it will allow it log the disc actions.
    8. Now, insert your SACD and have fun!

    The first decision to make is what format to rip in. However, a nearly complete consensus has emerged that rips should be done to ISO. The reasons for this are twofold: firstly that ripping to ISO is the only simple method to create a copy of the original SACD onto blank DVD media (sometimes called a SACD-R), and secondly that once you have an ISO to work with, the sacd_extract application can be used to extract and manipulate the audio data within the ISO - with no need to ever use the PS3 again for that SACD. Accordingly, you should rip to ISO unless you have some compelling reason not to.

    Process 1: Daemon Mode
    Sometimes called "Server Mode" or "Network Mode," this version may be simpler for those who have an ethernet connection to both the PS3 and the PC (and they are on the same network).

    Selecting no mode upon starting sacd-ripper will choose this mode by default after 5 seconds. This will allow you to automatically make SACD dumps over your local network. MAKE SURE AUTOMATIC UPDATES ARE DISABLED BEFORE CONNECTING YOUR PS3 TO THE NETWORK.

    Set into server mode, the Ripper Application will inform you of the PS3's IP address and the port it will be listening on. Insert a SACD into the PS3, and then move to the PC you will be using as a client for the rip. Then run sacd_extract to connect to the PS3 and commence the rip.

    Subject to changes in the application, the proper command to rip an ISO using the Daemon is sacd_extract -I -ixxx.xxx.xxx.xxx:xxxx (where the x-s represent the IP address and port that the Daemon displays on the screen. The ISO file will be placed in the same directory as sacd_extract. Note that unlike the PS3-centric ripper program, there is not a 4 GB file limit to rips done with the Daemon.

    Note that ripping using the Daemon may not produce a log, this is not unusual. However, do watch from the command line - sacd_extract will inform you as the rip progresses, and that the rip completed succesfully. If the program informs you of ANY read error, STOP the procedure and try to re-rip!

    Process 2: Ripping an ISO
    First of all, use "O" from the controller to navigate the ripping mode to ISO. Then use "X" to submit the ripping mode.

    The SACD-Ripper will then rip(and decrypt) the SACD that contains both Stereo and MultiChannel (If the SACD has it) onto the decrypted ISO. The ISO file name is depend on the original source folder name. If the ISO is bigger than 4GB, then there will be part files.

    Once done, you will have a ISO (or part files) and a sacd_log.txt as shown in the picture below:


    If you have the part files, you can use 7-zip to combine then easily. (Download 7-Zip here)

    Select the part files, File >> Combine



    As result, the ISO file will be combined to the destination you want.

    Make sure you attach the Log file from the ripping process and STOP the procedure if you see any read errors in the log!

    Process 3: Using SACD_extract

    Command line options explained: Show
    The following options are available for the sacd_extract commandline tool:

    Usage: sacd_extract [options] [outfile]
    -2, --2ch-tracks : Export two channel tracks (default)
    -m, --mch-tracks : Export multi-channel tracks
    -e, --output-dsdiff-em : output as Philips DSDIFF (Edit Master) file
    -p, --output-dsdiff : output as Philips DSDIFF file
    -s, --output-dsf : output as Sony DSF file
    -I, --output-iso : output as RAW ISO
    -c, --convert-dst : convert DST to DSD
    -C, --export-cue : Export a CUE Sheet
    -i, --input[=FILE] : set source and determine if "iso" image,
    device or server (ex. -i192.168.1.10:2002)
    -P, --print : display disc and track information

    Help options:
    -?, --help : Show this help message
    --usage : Display brief usage message


    Usage examples:

    Extract all stereo tracks to multiple DSDIFF files and convert all DST to DSD:

    sacd_extract -2 -p -c -i"Foo_Bar_RIP.ISO"

    Extract all multi channel tracks from the given ISO to multiple DSF files and convert all DST to DSD:

    sacd_extract -2 -s -i"Foo_Bar_RIP.ISO"

    Extract a single DSDIFF/DSD Multi-Channel Edit Master track from the given ISO and convert all DST to DSD:

    sacd_extract -m -e -c -i"Foo_Bar_RIP.ISO"

    Extract a single DSDIFF/DSD Stereo Edit Master track from the given ISO, create a CUE file, and convert all DST to DSD:

    sacd_extract -2 -e -c -C -i"Foo_Bar_RIP.ISO"

    Extract a single ISO file from the SACD Ripper Daemon (IP address and Port is displayed on startup). You can use SACD Extract again on the ISO file to extract the DSD data (see the four examples above):

    sacd_extract -I -i192.168.2.10:2002

    Extract all multi channel tracks from the SACD Ripper Daemon (IP address and Port is displayed on startup) to multiple DSDIFF files and keep the DST format:

    sacd_extract -m -p -i192.168.2.10:2002

    Generate a sacd_log.txt file that contains the ISRC codes which should/could be used for ISO verification.

    sacd_extract -P -i192.168.2.10:2002 >sacd_log.txt

    Process 4: Converting DSD to PCM
    Weiss Saracon allows you to make this conversion, which is necessary to upload to What.cd. (Get it here (Win) (OSX))

    What we will need is the DSD to PCM functionality from Saracon.

    Saracon has a very detailed logging window, you can copy-paste the job listings and use it as the log. To use the Saracon is quite easy.


    By selecting the DSD to PCM, and choose "Edit" to setup the conversion.


    Other than changing the Number Format to "24bit fixed point" and Sample Rate to "88.2KHz" or something "n x 44.1KHz" will be fine. Rest of them are pretty much default settings. If you would like to compress to FLAC directly without go through WAVE files yourself, you can do that by select Format as "FLAC". Drag the files into the DSD Input queue and you should be all set to go.



    Trim 0.0065s using software to take the static "click" noise off. You can use Saracon or other software such as Audition or Audacity.

    This sox command will do the job:
    sox <in.flac> <out.flac> trim 0.0065 reverse silence 1 0 0% trim 0.0065 reverse pad 0.0065 0.2


    After the conversion, copy the complete log out and paste it into a notepad, save it as a LOG file, include the PS3 log file as well.

    For playback of DSD files, you can use Foobar's DSDIFF plugin. (Info here) You can also use Foobar's re-sampler to convert the DSDIFF files to PCM and to FLAC, but apparently these are not always functional. Use Saracon if possible.

    Remember, these files aren't tagged. Make sure you tag them before uploading. A guide for tagging with foobar2000 can be found here.

    (Another method)
    If you are using a single DSDIFF file created using the method described above (in another words, one DSD file for the whole SACD), you should convert to .wav format need to use foobar2000 or another program to split the .wav and convert it to FLAC. Note that for multichannel files you will likely hit the 4 GB maximum size for a .wav file, in which case you should select .w64 as your file format. To split the .wav file edit the CUE file so it refers to the WAV file instead of the DSD file, and then open the CUE file in foobar2000 or another program which can open CUE files. Then convert the tracks to individual files using the same methods as described in the tutorial to convert DVD-Audio to FLAC in foobar2000 (except don't use the watermark detection tool). This method has a number of advantages, including the lack of a need to de-click each track, and the fact that tags will often be preserved (although they will probably need editing.
